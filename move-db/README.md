# move-db

SF6全キャラの技データ(ダメージ・補正値・フレーム等)をJSONで持つデータベース。
[damage-calc](../damage-calc/README.md) が毎回技データを聞き直さずに済むようにするのが目的。

## 現状(2026-07-23時点)

全30キャラの公式フレーム表をTSVに落としたところまで進んでいる(`raw-frame-tsv/`、2361技)。
JSONはまだ1件も無い。旧スキーマで作った`luke.json`があったが、スキーマを全面的に
書き直した際に実態と合わなくなったため削除し、TSVから作り直す方針にした。

## スキーマ(暫定)

定義は [schema.ts](schema.ts) にある。

- `baseDamage` / `starterScaling` / `comboScaling` / `immediateScaling` / `minGuarantee` は
  [damage-calc](../damage-calc/damage-calc.ts) の `MoveInput` にそのまま渡せる形にする
- 値がない/未確認の項目は `null`

## フレーム表HTMLのパース

公式サイトのフレーム表を、加工しやすいTSV/CSVに落とすパーサーが
[tools/parse-frame-html.ts](tools/parse-frame-html.ts) にある。

```bash
node --experimental-strip-types tools/parse-frame-html.ts raw-frame-html/luke.html > raw-frame-tsv/luke.tsv
node --experimental-strip-types tools/parse-frame-html.ts raw-frame-html/luke.html --csv
```

入力のHTMLは`raw-frame-html/`に、出力のTSVは`raw-frame-tsv/`に置く。公式サイトはSPAでフレーム表がJS描画のため、
URLを直接fetchしても表は取れない。ブラウザで
`https://www.streetfighter.com/6/character/<キャラ>/frame` を開き、DevToolsのElementsで
`<html>`のouterHTMLをコピーして保存する。日本語版・英語版のどちらでも動く。

出力は次の20列。**値の解釈は一切しない**。`※11`・`全体 47`・`22+着地後12`のような表記は
セルの文字列のまま出力する。数値化やスキーマへの写像は後段の別スクリプトの担当。

| 列                                      | 備考                                                           |
| --------------------------------------- | -------------------------------------------------------------- |
| キャラ / カテゴリ                       | カテゴリは表のセクション見出し(通常技・必殺技等)               |
| 技名 / 条件 / 入力                      | 公式表では1セル。条件は`（ジャンプ中に）`等の先頭のカッコ書き  |
| 発生 / 持続 / 持続内訳 / 硬直           | 持続内訳は多段技のみ(ツールチップに隠れているヒットごとの内訳) |
| ヒット硬直差 / ガード硬直差             |                                                                |
| キャンセル / ダメージ / コンボ補正値    |                                                                |
| Dゲージ増加ヒット / 減少ガード / 減少PC |                                                                |
| SAゲージ増加 / 属性 / 備考              |                                                                |

補足:

- 入力はアイコン画像なのでファイル名からテキストに戻している(`↓↘→ + 中P`、
  派生は`⇒`で繋ぐ)。レバーの`→`と紛らわしいので派生の矢印は別の字にしてある
- 列の対応付けはtdのCSSクラス名で行うため、列の順番が変わっても壊れない。ただし
  クラス名自体(`frame_startup_frame`等)が変わると壊れる
- セル内に複数項目がある場合(備考・コンボ補正値)は`|`で連結する
- 未知の入力アイコンが出てきたら`[ファイル名]`として出力し、stderrに警告を出す

## 対象キャラ

全キャラ。データ量が多いのでキャラ単位で区切って進める想定。

TSVは全30キャラ分ある。ヤスミンのみ配信前でフレーム表が存在しないため対象外。
JSON化はどのキャラも未着手。

## 保留事項

- **Dゲージ増減・SAゲージ増加量**: 公式フレーム表にはあるがスキーマにまだ項目がない。
  TSVには列として入っているので、スキーマが決まれば投入は可能。
- **飛び道具のhitstun/blockstun(絶対値)算出方法**: 発生・持続のフレームが弾の着弾タイミングと
  一致しないため、格闘ゲーム標準の「全体フレーム-持続終了フレーム=硬直」という単純計算が使えない。
  正しい算出方法が未確定なので、当面はサンドブラスト等の飛び道具技は`recovery`/`hitstun`/`blockstun`を
  `null`のままにしている。
- **多段技(1入力で複数ヒットする技)**: `hits`配列で表現できる設計にした(旧保留事項は解消)。
  ターゲットコンボ(ダブルインパクト等)は`derivesFrom`/`derivesInto`で繋いだ別々の技として
  表現する。
- **属性・キャンセル欄の表記揺れ**: TSVの素の値には次の揺れがある。JSON化時に解く必要がある。
  - 属性: `・`で1段内の複合(`上・弾`)、連結で段ごとの属性(`上中下中`)、`/`で条件分岐
    (`上・弾/上・空弾`)、先頭`※`で備考への注釈(`※中`)
  - キャンセル: 注釈の`※`が前置・後置・カンマ区切りのいずれもあり、位置に意味の違いはない。
    条件付きキャンセルか否かは備考の文面を読むしかない
- **飛び道具の行分割**: 公式表では弾を出す動作の行と弾そのものの行が分かれている
  (ディージェイのエアスラッシャー、ブランカのブランカちゃん爆弾等)。動作側の行は
  ダメージ0・発生なしになる。ディージェイの弱版のようにフェイントで弾が出ない技もある。
